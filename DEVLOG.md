- DECIMALS=-6, 같은 CIK의 다른 항목(민감도)은 DECIMALS=0으로 정상
- DART 공시에서도 단위가 "백만원"으로 표시 (본래 "원"이어야 맞음)

### 버그 수정: DECIMALS='INF' → IntCastingNaNError

아세아제지 DECIMALS 분포: `0`(3,296행), `INF`(173행), `-6`(137행), `7`(2), `6`(2)
- `pd.to_numeric('INF')` → `inf` → `.astype(int)` 에서 `IntCastingNaNError`
- 수정: `dec.where(dec.abs() < 100, 0)` 로 INF/비정상값을 0 치환 후 int 변환
- DECIMALS=0이면 `10^0=1`이라 VALUE 변동 없음 (안전)

### 4. 민감도 분석 → Duration/임금상승률 역산 설계

**발견**: 임금상승률 보유 + Duration 미보유 332사 중 163사(49%)가 민감도 TABLE 보유.

민감도 테이블 예시 (경동나비엔):
```
할인율      1%증가 → -4,345,783천원  |  1%감소 → +4,893,579천원
임금상승률   1%증가 → +4,979,626천원  |  1%감소 → -4,495,385천원
```

**역산 공식**: `Duration = |ΔDBO| / (DBO × 0.01)`
- 경동나비엔 검증: DBO 779억, 할인율 1%p 증가시 ΔDBO 43.5억 → Duration 5.58년
- 할인율 1%p 감소시 → 6.28년, 평균 5.93년 (convexity로 비대칭 정상)

**예상 커버리지 개선**:

| 항목 | 현재 | 민감도 역산 추가 | 순증 |
|------|------|----------------|------|
| Duration | 707 (42.1%) | **862 (51.4%)** | +155사 |
| 임금상승률 | 985 (58.7%) | 989 (58.9%) | +4사 |

**LLM 프롬프트 추가 예정 필드**:
- `SensitivityDR_1pct`: 할인율 1%p 증가시 DBO 변동액

- 2021→2022: ratio 0.504 — 금리급등(2.9%→5.3%) 구조적 특성 (IC는 기초 할인율 적용)

### 7. MacDur vs ModDur 분석

**시도**: `r = MacDur / ModDur - 1` 로 할인율 역산
**결과**: 체계적 과대 추정 (+7~8%p)
**원인**: 공시 Duration(가중평균만기) ≠ Macaulay Duration
- 공시: `Σ(t × CF_t) / Σ(CF_t)` — 명목 현금흐름 가중
- MacDur: `Σ(t × PV(CF_t)) / Σ(PV(CF_t))` — 현재가치 가중
- 공시 Duration > MacDur → 비율이 1+r 관계를 만족하지 않음

### 8. 부채증가율 모델 검토

**전임자 Vasicek 평균회귀 모델**:
```
R_(T+1) = R_T + α(R_∞ - R_T) + σW
I_(T+1) = I_T + β(I_∞ - I_T) + π(ρW + √(1-ρ²)Z)
```
- R: 할인율, I: 임금상승률, ρ: 상관계수
- 시나리오 테이블과 본질적으로 같은 식, 차이는 ΔR/Δg를 확률적으로 생성하는지 여부
- **목표수익률 산출 목적이면 시나리오 테이블로 충분**, Vasicek은 불확실성 범위 필요 시
